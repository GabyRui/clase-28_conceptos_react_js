Definición y ejemplo práctico de los 11 conceptos vistos y explicados en clase

PREGUNTA 1: ¿Qué es templateLiterals?

Es una característica de Javascript que permite crear cadenas de texto de manera versátil. En react se crean utilizando comillas invertidas.
Ver código en Javascript

PREGUNTA 2: ¿Qué es Short Hand Property?

Es una característica de Javascript que permite la asignación de valores a las propiedades de un objeto de forma legible. También es conocida como propiedades abreviadas,y es la forma más simple de definir objetos literales.
Ver código en Javascript

PREGUNTA 3: ¿Qué es Arrow Functions?

Llamadas también "funciones flecha" y permite que se disponga de una forma más concisa de escribir funcionas expresivas. Utiliza la sintaxis => para definir un Arrow Function.
Ver código en Javascript

PREGUNTA 4: ¿Qué es Desestructuring?

Permite la extracción de los valores de objetos y arreglos de forma más simplificada. La desestructuración permite un código más simple y legible, sobre todo cuando se aprecian estructuras con una mayor nivel de complejidad.
Ver código en Javascript

PREGUNTA 5: ¿Qué es Parameters Default?

Los parámetros por defecto hace referencia a la capacidad de asignar valores a los parámetros de una función en caso de que estos no dispongan de un valor inicial. De esta forma el código es más flexible, y la forma de definir los parámetros se da en la declaración de la función.
Ver código en Javascript - module_(parametersDefault)

PREGUNTA 6: ¿Qué es esModules?

Proporciona un sistema de módulos nativo, lo cual permite la reutilización de código mediante la importación de sus elementos. Los módulos permiten la división del código, y ello facilita la organización y manejo del mismo.
Ver código en Javascript

PREGUNTA 7: ¿Qué es ternaries?

También llamado como expresiones terniarias y es una construcción condicional abreviada y permite que la expresión condicional sea más eficiente, legible y simplificada.
Ver código en Javascript - Ternaries

PREGUNTA 8: ¿Qué es ArrayMethods?

Son funciones que permiten realizar operaciones en los arreglos. Permiten la manipulación de arreglos, la cual es una herramienta esencial para trabajar con la información en forma de arreglos, lo cual hace que el código sea más simple y conciso.
Ver código en Javascript - arrayMethods

PREGUNTA 9: ¿Qué es nullishOperator?

Permite proporcionar un valor de respaldo cuando se ubica un valor nulo o que está como undefined. Se aplica cuando se desea obtener el valor de una variable, o en caso contrario, se tiene un valor de respaldo. El valor de respaldo solo es proyectado siempre y cuando la variable es null o undefined, pero no si es una cadena vacía o tiene como valor false.
Ver código en Javascript - nullishOperator

PREGUNTA 10: ¿Qué es optional Chaining?

Este operador está representado por ? y permite el acceso a los elementos de un objeto de forma más segura. Este operador hace posible la verificación sobre si una propiedad está definida, en lugar de mostrar error, y en caso contrario de no estar definida, el resultado será mostrado como undefined.
Ver código en Javascript - optionalChaining

PREGUNTA 11: ¿Qué es async?

Se utiliza para la definición de funciones asincrónicas, las cuales son importantes para el manejo de operaciones asincrónicas de forma más legible y simplificada y son utilizadas para el desarrollo de sitios web y de app móviles. También pueden realizar tareas que pueden llevar tiempo sin bloquear la ejecución del código, y son muy útiles para la realización de operaciones, como las solicitudes API, lectura de archivos, y cualquier labor que comprenda la espera de que ocurra un evento. Al realizar una función Async, esta inmediatamente devolverá una promesa, la cual representa un valor que puede estar disponible en cualquier momento, tanto a corto como a largo plazo.
Ver código en Javascript - async
